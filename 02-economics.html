Object · 看清对象

Skill 是什么 —— 一种商品

Skill(技能)在 Claude 的体系里是一段带 schema 的提示词 + 工具声明 + 调用入口, 在 GPT 的体系里是一份 instructions.md + 一组 tools, 在 MCP 里是一段对外暴露的服务描述。

但从经济学的视角看,Skill 是一种商品 —— 它有作者、有买家、有价格、有被淘汰的宿命。

// Anthropic
prompt + tools + entry
// OpenAI

// MCP

Skill 的供给曲线有一个奇怪的特点:
接近零边际成本。

— Supply Side · 自然垄断的候选

写一份 Skill,前几个小时沉没成本高,但复用一万次,作者每多卖一份的成本几乎为零。

Demand · 需求侧三层

需求分层裂开

// L1 · to-C 尝鲜层

随便试一下,粘性极低。

价格弹性: 极高

// L2 · to-P 提效层

把 Skill 嵌进工作流,愿意为省时间付钱。

在意 ROI

// L3 · to-B 系统层

Skill 是组织内部"角色卡片",被采购、被治理。

在意合规

// 关键洞察

三层完全不同的人在想,他们的"价格弹性"差异巨大。

一份 Skill 同时卖三层?几乎不可能。

这是 Skill 经济的根本性未解问题。

Key Points · 三件非显然的事

关于 Skill 经济

  • 01
    零边际成本 ≠ 零价值
    一份 Skill 的稀缺资源不是"再次使用"的成本,而是"作者时间"和"信用" —— 这两件事都不是无限可复制的。
  • 02
    to-C / to-P / to-B 是三种产品
    同一份 Skill 想打通三层,几乎一定失败。版本必须分裂,定位必须切割。
  • 03
    "被用"与"被看见"已脱钩
    一个 Skill 装进 5 万个工作流,作者可能完全不知道 —— 传统的 star / download 反馈环失效了。
Limit · 这副镜片的盲区

经济学看不到什么

// 它擅长看的

  • 供需、激励、抽成、替代、互补
  • 平台与创作者的博弈
  • 需求分层与价格弹性

// 它看不到的

  • 作者为什么写 —— 动机不仅是钱
  • Skill 的"语义价值"如何定价
  • 信任、信用、声誉这些"非货币变量"

经济学 + 关系视角 + 制度视角,合用才完整。

Judgment · 一个判断

我赌:Skill 不会像 App 一样
爆炸成"千亿市场"。

它更可能像Linux 包管理——
重要但不性感,被少数人维护,被多数人白嫖。

真正赚钱的,不是 Skill 本身,
而是 Skill 嵌入的工作流。
