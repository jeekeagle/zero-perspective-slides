— Core Insight · 拓扑决定性质

同样的 agent 能力,放在不同的拓扑里,
会做出完全不同的系统。

Properties · 拓扑与性质

同样的 agent,不同的代价

// Pipeline

可控、稳定、易调试

代价:串行延迟 = 各环节延迟之和

// Fan-out

快(并行)、可对比(多视角)

代价:成本 = N 倍

// Loop(反思)

质量高(自我修正)

代价:不可预测完成时间,可能死循环

// Mesh

最灵活

代价:调试地狱、状态爆炸

Supervision · 监督关系的拓扑

组织学的老问题,新版本

  • 01
    监督关系画出来是树
    每个节点最多一个父,但可以有多个子。换成人:每个员工一个老板,可以管多人。
  • 02
    层级数 = 0(Mesh)→ 乱
    没人拍板,谁都能调谁。在人类社会这叫"无政府",在 agent 系统这叫"调试噩梦"。
  • 03
    监督者是否同质
    同质 → 容易被群体偏见带偏;异质 → 慢但稳。组织管理学的核心矛盾,搬进了 agent 系统。
