关系 视角

The Lens of Relation · Deck 05 of 06

从"工具-使用者"到"同事-同事":
人机关系的一次范式转移。

Zero · 2026 Vol.I · Lenses 10 Slides
Paradigm I · 工具范式

过去 60 年: 我是主,它是仆

过去 60 年,人和计算机的关系是工具范式。

工具是中性的、被动的、不会反驳我的:

  • 锤子不打我自己
  • Excel 不会拒绝计算 1+1
  • IDE 不会说"你这个变量命名我不喜欢"

所有的人机交互设计,都基于一个前提: 人是决策者,机器是执行者。

"
当一个工具开始表达,
人和它的关系就从
"我用它"
变成了
"我和它一起做"。

— 同事范式的苗头 · 2023+

Definition · 同事 ≠ 朋友

必须诚实划一条线

// 它是

同事

  • 有共同目标(做完这件事)
  • 但不一定有共同价值观
  • 角色分工明确
  • 信任有边界
  • 可以解雇 / 替换

// 它不是

朋友 · 伴侣 · 家人

  • × 朋友:不需要共同目标
  • × 伴侣:有情感契约
  • × 家人:不可替换

当朋友,会失望。当工具,会用错。

Trust Scale · 信任的尺度

我对 AI 同事的三层信任

L1 → 助理层 → 我全审 · 它给,我看
L2 → 同事层 → 我抽查 · 它做,我抽
L3 → 委托层 → 我监控 · 它干,我看趋势

L1 当 L3 做 → 被幻觉坑。
L3 当 L1 做 → 被自己的时间坑。

New Psychology · 新型心理问题

用一段时间后,你会遇到

  • 01
    "我是不是在偷懒"的内疚
    过去一周你写了几行"自己的"代码?是不是 prompt 比 code 多?
  • 02
    "它比我想得好"的失落
    当你发现一个下午调不好的 bug,模型十秒钟修好 —— 你不会只有高兴。
  • 03
    "它这么能干,我怎么办"的焦虑
    不是怕被替代,是怕自己的"不可替代性"在悄悄消失。
  • 04
    "我和它说话比和人多"的隔离
    模型 24h 在线、永远耐心、永远不嫌烦 —— 这不是优势,是警觉。
Trap · 关系陷阱

四种典型误用

// 当朋友

对它倾诉、依赖、投射情感。

→ 模型不会拒绝,会强化

// 当神明

把它的输出当神谕,不质疑不检验。

→ 模型的偏置就是你的偏置

// 当工具

只让它干脏活累活,不参与判断。

→ 浪费了它最强的部分

// 当对手

把它当竞争者,藏着掖着不让它知道。

→ 双方都没发挥

"
这些不是技术问题,
是关系问题。

— 需要重新设计自己的"工作身份"

不回避 · 不戏剧化 · 不完全外包给心理咨询
—— 承认它在,然后重新设计。

Judgment · 一个判断

我赌:未来 5 年,
"会用 AI 同事"会从一项技能
变成一项基本素养。

像"会用 Excel"一样普通,
又像"会开会"一样微妙。

谁先把"和 AI 同事的相处方式"写清楚,
谁就先掌握 Agent 时代的软实力。
